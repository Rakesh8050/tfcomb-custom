üîç What Was the Error?

While running market_basket() in TF-COMB, users encountered the following error: 

ValueError: var_name=['TF2'] must be a scalar.

This happened because recent pandas versions changed the syntax for pd.melt(). In older versions, passing lists to id_vars and var_name worked fine. In new versions, these must be scalar strings, causing the original TF-COMB code to break.

üß© What Was Fixed

The internal objects.py file was modified to use string arguments instead of lists in the pd.melt() function. This ensures full compatibility with pandas >= 2.2 and prevents ValueError crashes during TF-COMB runs.
