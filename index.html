üîç What Was the Error?

When running market_basket() in TF-COMB, users encountered: 

ValueError: var_name=['TF2'] must be a scalar.

This occurred because newer versions of pandas changed how pd.melt() accepts parameters. In older versions, lists for id_vars and var_name worked fine. Now, they must be single string (scalar) values. TF-COMB‚Äôs original code used lists, causing incompatibility.

üß© What Was Fixed

The objects.py file inside TF-COMB was updated to replace list-based arguments with string-based ones in the pd.melt() call. This ensures compatibility with pandas ‚â• 2.2 and prevents the ValueError crash.
